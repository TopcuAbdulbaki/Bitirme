## KRİTİK TASARIM KARARLARI (hemen netleştirin)

| Karar Alanı | Önerim (neden) |
|-------------|----------------|
| **Topoloji** | **Yıldız + Message Broker**. Orchestrator merkezde, diğer node’lar sadece ona bağlanır → debugging, scaling, version upgrade kolay. |
| **İletişim Protokolü** | **gRPC + Protobuf** (binary, streaming, 3–5× düşük latency). HTTP/REST sadece GUI ↔ Orchestrator için. |
| **Asenkron Kuyruk** | **Kafka** (exactly-once, schema registry) veya **RabbitMQ** (daha hafif). Her node publish/subscribe yapar. |
| **Model Deployment** | **vLLM** (Qwen3-VL 8B için 1×A100, 30B-A3B için 2×A100 + tensor-parallel). Quantization: AWQ 4-bit. |
| **Crawler Seçimi** | **Firecrawl** (API + webhook, robots.txt uyumlu). Playwright sadece “deep-dive” fallback. |
| **Yasal Koruma** | Sadece **RSS + resmi API** + **robots.txt whitelist**. Tarayıcı modülüne rate-limit 1 req/5 sn + random user-agent. |
| **Veri Schema** | Protobuf mesajları:<br>`NewsItem { url, title, raw_html, images[], timestamp, source_type }`<br>`ProcessedItem { summary, entities, sentiment, category, image_captions[], embeddings[] }` |
| **Monitoring** | Prometheus exporters her node’da, Grafana dashboard, Loki log aggregation. |
| **Deployment** | **Docker Compose** dev için, **K8s (kind veya EKS/GKE)** prod için. Helm chart’ler hazır. |
