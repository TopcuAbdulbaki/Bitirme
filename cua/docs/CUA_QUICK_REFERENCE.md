## 🚀 Quick Start

### 1. Local Development (No GPU Required)

```bash
# Terminal 1: Orchestrator
python -m orchestrator.main

# Terminal 2: CUA Node (with stubs)
python -m cua.main

# Expected output from CUA:
# [CUA] Starting CUA Node...
# [CUA] Registered as cua_XXXXXXXX
# [CUA] Connected to RabbitMQ
# [CUA] Task consumer loop started
# [CUA] Heartbeat sent every 10s
```

### 2. Docker Deployment

```bash
# Build CUA image only
docker build -t abdulbakitopcu/cua:latest -f cua/Dockerfile .

# Run full stack (Orchestrator + CUA)
docker-compose up --build orchestrator cua

# Check CUA logs
docker logs cua-node
```

### 3. Remote Deployment (Vast.ai)

```powershell
# With SSH to Vast instance
.\scripts\legacy\6_cua.ps1 -RemoteHost <IP> -RemoteUser root -UseSSH $true

# Expected output:
# [CUA] Building Docker image...
# [CUA] Deploying to remote host via SSH...
# [CUA] CUA node deployed to <IP>
```

## 🔧 Configuration

### Environment Variables (in `.env` or docker-compose)

```bash
# Orchestrator
ORCHESTRATOR_HOST=localhost          # localhost or service name
ORCHESTRATOR_PORT=50051
CUA_GRPC_PORT=50054

# RabbitMQ
RABBITMQ_HOST=rabbitmq              # localhost or service name
RABBITMQ_PORT=5672
RABBITMQ_USER=guest
RABBITMQ_PASSWORD=guest

# LLM Model
MODEL_MODE=local                     # "local" veya "production"
MODEL_NAME=Qwen/Qwen3.5-9B-Instruct  # Production için
LMSTUDIO_URL=http://localhost:8000/v1  # Vast.ai vLLM endpoint (local dev: 1234)

# Agent Parameters
MAX_ARTICLES=10                      # Surface mod: kaç makale bulunacak
MAX_SEARCHES=5                       # Max arama iterasyonu
SEARCH_ENGINE=duckduckgo             # "duckduckgo" (önerilen, CAPTCHA yok) | "bing" | "google"
CONFIDENCE_THRESHOLD=0.80            # Research mod: durma eşiği
MAX_RESEARCH_CYCLES=15               # Max araştırma döngüsü
SEARCH_DELAY=1.5                     # Aramalar arası bekleme (saniye)
```

## 📊 Task Types

### Mode 1: Surface Search (Fast News Discovery)

```python
# Orchestrator queues:
{
    "mode": "surface",
    "query": "turkey economy news",
    "params": {
        "max_articles": 10,
        "max_searches": 5
    }
}

# CUA runs: plan → execute (search) → evaluate (count) → synthesize
# Output: List of 10 articles in crawler-compatible format
# Result: Each article fed into VLM/LLM pipeline
```

### Mode 2: Deep Research (Intelligence Analysis)

```python
# Orchestrator queues:
{
    "mode": "research",
    "topic": "2026 turkey economic crisis predictions",
    "params": {
        "max_cycles": 15,
        "confidence_threshold": 0.80
    }
}

# CUA runs: plan → execute (search/visit) → evaluate (confidence) → synthesize
# Output: Comprehensive report with findings and hypothesis
# Result: Stored in research_missions table with mission_id
```

## 🗄️ Database Schema

### New Table: `research_missions`

```sql
CREATE TABLE research_missions (
    mission_id VARCHAR(64) PRIMARY KEY,
    topic TEXT NOT NULL,
    status VARCHAR(20) DEFAULT 'in_progress',  -- in_progress, completed, failed
    final_report_json JSONB,                   -- Synthesized report
    graph_state_json JSONB,                    -- LangGraph final state
    findings_count INTEGER DEFAULT 0,
    confidence_score FLOAT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP
);

-- Index for queries
CREATE INDEX idx_research_status ON research_missions(status);
```

### New Columns: `news` table

```sql
ALTER TABLE news ADD COLUMN source_type VARCHAR(20) DEFAULT 'crawler';
-- Values: 'crawler' (default), 'agent_surface', 'agent_research'

ALTER TABLE news ADD COLUMN mission_id VARCHAR(64);
-- Foreign key to research_missions.mission_id (nullable)

CREATE INDEX idx_news_mission ON news(mission_id);
```

## 🔌 Integration Points

### 1. **Orchestrator → CUA** (gRPC)
- CUA registers: `RegisterRequest(node_type="cua", host="...", port=50054)`
- CUA heartbeat: `Heartbeat(node_id="cua_XXXXXXXX")` every 10s
- Result: Node shows in `status` command

### 2. **Orchestrator → CUA** (RabbitMQ)
- Publish task: `publish_agent_task(task_id, json_data)`
- CUA consumes: `consumer.get_message("agent_tasks")`
- CUA publishes result: `publisher("agent_results", result)`

### 3. **CUA → Database** (gRPC)
- Mode 1: `StoreDataRequest` with articles (→ news table)
- Mode 2: `insert_research_mission`, `complete_research_mission`

### 4. **CUA → Pipeline** (Automatic)
- Surface mode articles → VLM analysis → LLM sentiment → final storage
- Deep research → Stored in research_missions table

## 🐛 Troubleshooting

### "CUA fails to register"
```
Error: [CUA] Registration failed: ...
Fix: 
  1. Check ORCHESTRATOR_HOST and ORCHESTRATOR_PORT
  2. Verify orchestrator is running: docker logs orchestrator
  3. Verify gRPC port 50051 is accessible
```

### "RabbitMQ connection failed"
```
Error: [CUA] RabbitMQ connection failed: ...
Fix:
  1. Check RABBITMQ_HOST and RABBITMQ_PORT
  2. Verify rabbitmq container: docker ps | grep rabbitmq
  3. Check credentials: RABBITMQ_USER=guest, RABBITMQ_PASSWORD=guest
```

### "LangGraph state transition error"
```
Error: StateGraph ValueError: ...
Fix:
  1. Check AgentState TypedDict has all required fields
  2. Verify plan_node returns dict with updated state
  3. Check should_stop flag is boolean
```

### "Docker build fails (Chromium)"
```
Error: Dockerfile build fails at playwright install chromium
Fix:
  1. Ensure NVIDIA Container Toolkit is installed: nvidia-ctk --version
  2. Use nvidia-docker instead of docker
  3. Check internet connectivity during build
```

### "GPU not available"
```
Error: torch.cuda.is_available() returns False
Fix:
  1. Check docker-compose GPU config: "driver: nvidia", "count: 1"
  2. Verify NVIDIA Docker: docker run --rm --gpus all nvidia/cuda:12.1.0-runtime-ubuntu22.04 nvidia-smi
  3. On Vast.ai: ensure instance has GPU selected
```

## 📈 Monitoring

### Node Health
```bash
# Check CUA registration
docker logs orchestrator | grep "cua_"

# Check heartbeat
docker logs orchestrator | grep "Heartbeat.*cua_"

docker logs cua-node | grep "Task received"
```
